Slide 7: Variables in Python - Naming and Assignment

Instructions:

Explain variable naming conventions (snake_case) and dynamic typing.

Variable Naming Conventions: In Python, we typically use snake_case for variable names. This means words are separated by underscores (_). For example: my_variable, user_name, total_count. Variable names should be descriptive and start with a letter or underscore. They cannot start with a number. Case matters (myVariable is different from myvariable).

Dynamic Typing: Python is dynamically typed, which means you don't need to explicitly declare the data type of a variable. The interpreter infers the type based on the value assigned to it. This makes Python code more concise and readable.

Example:

        
            name = "Alice"  # Python infers 'name' is a string
            age = 30       # Python infers 'age' is an integer
            height = 5.9  # Python infers 'height' is a float
            is_student = True # Python infers 'is_student' is a boolean
