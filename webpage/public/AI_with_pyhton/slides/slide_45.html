Slide 45: Virtual Environments with venv/virtualenv

Why Use Virtual Environments?

Virtual environments provide isolated spaces for Python projects. This means:

  • Dependency Management: Different projects can use different versions of the same package without conflicts.
  • Project Isolation: Keeps your project's dependencies separate from the global Python installation, preventing accidental modification of system-wide packages.
  • Reproducibility: Ensures that your project can be easily replicated on different machines with the exact same dependencies.
  • Clean Environment: Allows you to experiment with different packages without cluttering your system's Python environment.

Creating a Virtual Environment with venv

venv is Python's built-in virtual environment module (available since Python 3.3). Here's how to use it:

  1. Navigate to your project directory: 
    cd your_project_directory
  2. Create the virtual environment: 
    python3 -m venv .venv

    This creates a directory named .venv (or any name you choose) to hold the environment.

  3. Activate the environment:
    • Linux/macOS: 
      source .venv/bin/activate
    • Windows: 
      .venv\Scripts\activate

    Your shell prompt should now be prefixed with the environment name (e.g., (.venv)).

  4. Install packages: 
    pip install requests
  5. Deactivate the environment: 
    deactivate

Creating a Virtual Environment with virtualenv

virtualenv is an older, third-party virtual environment tool. While venv is generally preferred, virtualenv might be necessary in some legacy situations or if you need to support older Python versions.

  1. Install virtualenv: 
    pip install virtualenv
  2. Navigate to your project directory: 
    cd your_project_directory
  3. Create the virtual environment: 
    virtualenv venv

    This creates a directory named venv (or any name you choose) to hold the environment.

  4. Activate the environment:
    • Linux/macOS: 
      source venv/bin/activate
    • Windows: 
      venv\Scripts\activate

    Your shell prompt should now be prefixed with the environment name (e.g., (venv)).

  5. Install packages: 
    pip install requests
  6. Deactivate the environment: 
    deactivate
